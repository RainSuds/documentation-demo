Quickstart for GitHub Pages

This guide outlines the steps to quickly set up a GitHub Pages website. It is designed for developers and teams seeking an efficient way to publish project documentation, portfolios, or blogs.

Intended Audience

This documentation is intended for users familiar with basic GitHub operations who want a streamlined process for deploying their site. Ensure you have an active GitHub account and the necessary permissions to create and manage repositories.

Note: GitHub Pages uses GitHub Actions for site builds (commonly using Jekyll). If you prefer not to use Jekyll, simply add a .nojekyll file to your repository.

Overview

GitHub Pages allows you to host static websites directly from your GitHub repository. This guide will show you how to create a site using a pre-built theme, customize it, and publish your content at username.github.io.

Creating Your Website

  1. Click the + icon in the upper-right corner of GitHub and select New repository.
  2. Name your repository username.github.io (replace username with your GitHub handle, e.g., octocat.github.io).
  3. Choose your repository’s visibility and select Initialize this repository with a README.
  4. Click Create repository to complete the setup.
  5. Navigate to the repository’s Settings tab. If the tab isn’t visible, open the dropdown menu and select Settings.
  6. In the sidebar under Pages, set the source to deploy from a branch by selecting your desired branch (typically main) and folder (/ or /docs).
  7. After saving the settings, wait a few minutes for GitHub to build and publish your site.
  8. Visit your site at https://username.github.io (replace username with your actual GitHub username) to see it live.

Changing the Title and Description

By default, your site’s title is set to username.github.io. To customize the title and add a description, follow these steps:

  1. Click the Code tab in your repository.
  2. Open the _config.yml file from the file list.
  3. Click the edit icon (pencil) to modify the file.
  4. Add the following lines (or update them) under the existing theme setting:
    5. theme: jekyll-theme-minimal
title: My Awesome Homepage
description: A hub for all my project updates and documentation.
  5. Click Commit changes when you’re finished editing.
